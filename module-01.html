1교시. 배포관리 개요와 프로젝트 구조

학습 목표
  • 소프트웨어 배포·업데이트·마이그레이션의 개념을 설명할 수 있다
  • 본 프로젝트(E-updateSystem)의 전체 아키텍처를 그릴 수 있다
  • 개발 환경을 구축하고 빌드할 수 있다
이론

1-1. 왜 배포관리가 필요한가? (15분)

수백~수천 대의 PC·키오스크·산업용 단말에 동일한 버전의 프로그램과 설정을 유지해야 할 때, 수작업 복사는 실수·버전 불일치·보안 사고로 이어집니다.

  • 배포(Deploy): 승인된 패키지를 대상에 전달·설치
  • 업데이트(Update): 실행 파일·설정 등 버전 갱신
  • 마이그레이션(Migration): DB 스키마·설정 형식 변경 적용

본 프로젝트는 중앙 Flask 관리 콘솔과 C 언어 배포 에이전트로 이 과정을 자동화합니다.

실습

★ 실습 1: 환경 구축 및 빌드 (20분)

배포 서버는 Ubuntu에 두고, 클라이언트는 Ubuntu·Windows 등에 설치하는 구성을 가정합니다.

  1. 프로젝트를 Ubuntu 서버에 배치 (예: /opt/E-updateSystem)
  2. sudo apt install build-essential python3-pip python3-venv
  3. Ubuntu 서버·Linux 클라이언트: make → deploy_server/deploy_server, deploy_client/deploy_client
  4. Windows 클라이언트만: build.bat → .exe 생성 (서버는 Ubuntu 유지)
  5. Flask: cd flask_app && pip install -r requirements.txt && python3 app.py
  6. 브라우저에서 강의·대시보드 접속 확인
체크리스트
  • □ Ubuntu: deploy_server/deploy_server 빌드됨
  • □ 클라이언트 OS에 맞는 바이너리 준비됨
  • □ Flask 대시보드 열림

# ============================================================
# Makefile — Ubuntu/Linux 에서 배포 서버·클라이언트 빌드
# 사용: make          (빌드)
#       make clean    (생성 파일 삭제)
# Windows 클라이언트만 build.bat 사용
# ============================================================

CC ?= gcc
CFLAGS = -Wall -Wextra -O2 -Ideploy_common -pthread
# 여러 .c 파일을 링크할 때 공통으로 쓰는 소스
COMMON = deploy_common/config.c deploy_common/thread_pool.c \
         deploy_common/package_cache.c deploy_common/ntp_sync.c
LDFLAGS = -pthread

all: dirs deploy_server/deploy_server deploy_client/deploy_client packages

dirs:
	@mkdir -p packages deploy_client/installed 2>/dev/null || true

# 배포 서버 실행 파일
deploy_server/deploy_server: deploy_server/server.c $(COMMON)
	$(CC) $(CFLAGS) -o $@ deploy_server/server.c $(COMMON) $(LDFLAGS)

# 배포 클라이언트 실행 파일 (Ubuntu PC 또는 Windows용 WSL에서 빌드)
deploy_client/deploy_client: deploy_client/client.c $(COMMON)
	$(CC) $(CFLAGS) -o $@ deploy_client/client.c $(COMMON) $(LDFLAGS)

# 데모용 패키지 파일 생성
packages:
	@echo demo > packages/app.bin
	@echo '[config]' > packages/config.ini
	@echo 'version=1.0.1' >> packages/config.ini
	@echo '-- migrate' > packages/migrate_v2.sql
	@echo '{"patch":2}' > packages/schema_patch.json

clean:
	rm -f deploy_server/deploy_server deploy_client/deploy_client
